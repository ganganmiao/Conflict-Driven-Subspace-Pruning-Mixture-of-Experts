## 实验报告：纯盲基线 (Pure Blind Baseline) 对比实验分析

为了严格验证 CDSP 架构中的语义涌现是否依赖于冲突博弈机制，而非单纯源于数据本身的统计分布差异，我们引入了 **纯盲训练基线 (Pure Blind Baseline)** 作为对照组。

在该对照实验中，模型架构与 CDSP 保持参数一致，但**全程移除 Task ID**，仅依靠负载均衡损失（Aux Loss）和图像内容进行训练。我们旨在观察：在没有冲突惩罚的情况下，仅凭数据统计差异，模型能否自发实现清晰的语义分工？

### 1. 演化轨迹：从赢家通吃到语义纠缠

通过追踪纯盲基线在不同训练阶段的路由分布，我们观测到了典型的 **“赢家通吃 (Winner-Take-All)”** 与 **“语义纠缠 (Semantic Entanglement)”** 现象。

* **初始偏置阶段 (Epoch 0)**:
* **现象**: 初始化伊始，系统并未表现出均匀分布，而是迅速对 **E6 (0.32)** 和 **E7 (0.21-0.35)** 表现出强烈偏好。
* **特征**: 此时 Task 0 (MNIST) 和 Task 1 (KMNIST) 的路由指纹高度相似，而 Task 2 (FMNIST) 虽然在 E7 上的权重略高，但整体活跃专家集合与前两者完全重叠。


* **中期固化阶段 (Epoch 3)**:
* **现象**: 路由格局开始固化，**E0** 的地位显著上升，与 **E6**、**E7** 共同构成了“寡头集团”。
* **纠缠**: 值得注意的是，三个任务在非活跃专家（如 E3, E4）上的权重均被压缩至极低（0.01-0.02），而在活跃专家上的分布呈现出**高度的共享性**。Router 并没有为 Task 2 开辟新的“领地”，而是让它继续挤在 E0/E6/E7 的共享空间里。


* **终局静态阶段 (Epoch 9)**:
* **现象**: 系统进入最终稳定态。
* **Task 0**: E0 (0.21) + E5 (0.20) + E6 (0.29)
* **Task 1**: E0 (0.22) + E5 (0.16) + E6 (0.29)
* **Task 2**: E0 (0.31) + E5 (0.11) + E6 (0.21) + E7 (0.29)


* **结论**: 尽管 Task 2 在 E7 上的权重（0.29）高于其他任务，但它**同时大量占用了** Task 0/1 的主力专家 E0 (0.31) 和 E6 (0.21)。这表明模型**未能实现语义隔离**，而是采用了一种“混合共享”策略来降低整体 Loss。

### 3. 核心对比结论：冲突即结构

通过分析上述数据，我们得出以下关键结论：

1. **严重的语义重叠 (Semantic Overlap)**:
观察上表，**E0, E6, E7** 是所有三个任务共同争夺的“热点资源”。
* **Task 0/1** 主要依赖 E6 (0.28/0.29) 和 E0 (0.24)。
* **Task 2** 虽然是完全不同的视觉任务（实物），却依然将 **34%** 的流量导向了 E0，**20%** 的流量导向了 E6。
* 这证明 Baseline **未能将“符号识别”与“物体识别”在物理空间上隔离开**。


2. **Aux Loss 导致的伪均衡**:
Baseline 虽然通过 Aux Loss 避免了单专家坍塌（Single Expert Collapse），但它陷入了另一种**“平均主义陷阱”**。为了满足负载均衡，它强行让 Task 2 去使用 Task 0 的专家（E0, E6），导致专家无法特化。
3. **CDSP 的优越性**:
回顾 CDSP 的实验结果，Task 2 被清晰地隔离在 **E7**（独占），而 Task 0/1 聚类在 E0/E5。对比之下，Pure Blind Baseline 的结果是一团**“你中有我，我中有你”的混沌**。这有力地证明了：**仅靠数据统计差异不足以驱动清晰的模块化分工，冲突博弈机制是打破对称性、实现语义解耦的必要条件。**
